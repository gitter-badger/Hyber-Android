1) Приложение регистрирует в google developer consol, собирает и публикует Клиент:

	- Клиент должен зарегистрировать ключи для GCM (android, iOS, server);

	- Клиент должен предоставить для GMS, project Id, GCM server key;

	- Клиент должен получить от GMS ключ доступа к OTT;

	- После всего, Клиент подставляет свой project Id и ключ доступа к OTT в SDK и публикует приложение в маркет.


2) Приложение регистрирует в google developer consol, собирает и публикует GMS:

	- Клиент должен предоставить GMS доступ к google developer consol или создать новую и дать полный доступ Клиенту;

	- GMS должен зарегистрировать ключи для GCM (android, iOS, server) и сгенерировать конфигурационнай файл google-services.json;

	- GMS должен зарегистрировать аккаунт Клиента в OTT;

	- GMS должен подключить project Id и ключ доступа к OTT в SDK, подключить в проект конфигурационнай файл google-services.json и опубликовать приложение в маркете от имени Клиента.


3) Приложение собирает и публикует Клиент, а регистрирует в google developer consol GMS:

	- Клиент должен предоставить GMS доступ к google developer consol или создать новую и дать полный доступ Клиенту;

	- GMS должен зарегистрировать ключи для GCM (android, iOS, server) и сгенерировать конфигурационнай файл google-services.json;

	- GMS должен зарегистрировать аккаунт Клиента в OTT;

	- Клиент должен получить от GMS ключ доступа к OTT и конфигурационнай файл google-services.json;

	- Клиент должен подключить project Id и ключ доступа к OTT в SDK, подключить в проект конфигурационнай файл google-services.json и публикует приложение в маркет.


4) Приложение собирает и публикует GMS, а регистрирует в google developer consol Клиент:

	- Клиент должен зарегистрировать ключи для GCM (android, iOS, server);

	- Клиент должен предоставить для GMS, project Id, GCM server key и конфигурационнай файл google-services.json (для android) (или предоставить для GMS доступ разработчика к аккаунту google developer consol);

	- GMS должен зарегистрировать аккаунт Клиента в OTT;

	- GMS должен подключить project Id, ключ доступа к OTT в SDK, подключить в проект конфигурационнай файл google-services.json и опубликовать приложение в маркете от имени Клиента.


5) Приложение собирает GMS, а публикует Клиент (и наоборот):

	- Лучше не надо.
